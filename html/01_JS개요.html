프로그래밍 언어

    프로그램(program)
    - 명령어의 집합

    프로그래밍(programming)
    - 명령어를 나열하는 것 == 명령어 작성(코딩)

    프로그래머(programer)
    - 명령어를 작성하는 사람
    (참고) 개발자(devleoper)
    - 기획, 설계, 코딩, 테스트, 유지보수등 개발과 관련된 모든일을 하는 사람

    프로그래밍 언어(programming language)
    - 컴퓨터한테 명령어를 전달할 수 있는 언어
    - 개발자가 알아볼 수 있는 형태로 작성(형태/규칙이 정해져 있음)
      -> 컴퓨터가 해석할수 있는 형태(2진법)로 변환
      -> 명령어를 연산/수행

JavaScript

Script 언어

    기본 프로그램의 동작을 사용자의 요구에 맞게 수행되도록 해주는 언어(일괄 번역)
    별도의 소스코드 컴파일(한 번에 2진법 변환)을 하지 않고 인터프리터를 이용해 한 줄씩 번역, 수행하는 언어(실시간 번역)
    - 장점 : 컴파일에 소요되는 시간이 없음
    - 단점 : 프로그램 실행 시간이 상대적으로 느림

JavaScript 란?

    웹 브라우저에서 주로 사용하는 인터프리터 방식의 객체지향 프로그래밍 언어

    ECMA(European Computer Manufacturers Association)
    Script 표준을 따르는 대표적인 웹 기술(JS 버전면 : ES5,ES6, ...)

JavaScript 작성 방법

  1. 브라우저 개발자 도구 -> Console에 입력하기
  2. HTML 문서 내부에 작성하는 방법(script) -> Internal 방식

  3. HTML 문서 외부에 존재하는 .js 파일을 연결해서 사용 -> External 방식 
            1) 원하는 위치에 . 파일 생성
        2) 연결을 원하는 html 파일 body 제일 아랫쪽에 script를 작성 -> src 속성 추가
  4. 태그에 직접 JS 코드를 작성 -> inline 방식(권장 X)
