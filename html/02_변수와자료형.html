변수(Variable)

    - 변수 : 메모리 값을 저장할 수 있는 공간
      (RAM, 메모리 : 주기억장치
      - RAM에 기록된 값, 코드만 프로그램 실행 사용 가능)
    
      - 변수에 원하는 이름을 붙이고 값을 대입(=)
      
      - 필요한 곳에서 '변수명'을 호출하면 해당 변수에 저장된 값을 가져와 사용 가능

      - 변수에 저장된 값은 언제든지 변경 가능(단, 상수는 제외)

변수명 명명 규칙

    - 의미 있는 이름을 작성(임시명 a,b 이런 형태는 지양)
    - 대소문자 구분
    - 특수 문자는 _(언더스코어)만 사용 가능(단, 첫 글자는 제외)
    - 숫자도 사용 가능(단, 첫 글자는 제외)
    - Camel Case(카멜 표기법)으로 작성 -> 여러 단어를 연달아 작성하는 경우
      1) 기본적으로 소문자
      2) 연결되는 단어의 첫 글자만 대문자로 작성

변수의 종류와 특징

    var   : 변수(값 변경 ok), 변수명 중복 ok, 함수   레벨 범위
    let   : 변수(값 변경 ok), 변수명 중복 X,  블록{} 레벨 범위
    const : 상수(값 변경 X),  변수명 중복 X,  블록{} 레벨 범위

    * 변수 선언 방법
      var   변수명;
      let   변수명;
      const 변수명;

범위(scope)

JavaScript 자료형(Date type)

    * Date Type : Date(값)의 Type(형태)를 구분하는 것
      - 값의 연간 결과에 따른 동작 구분 or 오류 처리, 기능 선택 등..
    * JavaScript는 변수 선언 시 별도의 자료형을 지정하지 않음
      단, 변수에 값이 대입되어 질때 자료형이 결정된다.
    
    - strong(문자열)  -> "", ''형태의 값(리터럴)
    - number(숫자)    -> 정수, 실수 형태의 값(리터럴)
    - boolean(논리값) -> true(참), false(거짓)
    - object(객체)    -> 여러 값을 저장하는 형태 ex) JS 객체 {k:v, k:v, k:v}
    - function(함수)  -> 함수도 변수에 저장 가능
    - undefined(정의되지 않은 변수) -> 변수가 선언만 되어있는 상태
    
    (참고)
    - null(참조하는게 없음) -> 객체를 찾는 코드가 대입 되어있지만 객체가 없음을 의미
