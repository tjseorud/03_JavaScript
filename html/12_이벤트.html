Event

    * Event : 사건(행동, 행위, 동작) -> 브라우저에서 할 수 있는 동작
      ex) click, keydown, keypress, keyup, change, submit, focus, blur, drag,
          mouseover, mouseout, mouseon, mouselaeve, ...

    * Event Listener(이벤트 리스너) : 
      - 브라우저에서 발생한 이벤트 감지
      - 이벤트를 감지 될 때 까지 대기, 감지 되었을 때
      - 이벤트 리스너 추가 방법
        1) on'EventName'
          ex)onclick, onkeyup, onchange, ...
        2) 요소.addEventListener('Event',event);
      
    * Event Handler(이벤트 처리기, 핸들러)
      - 이벤트 리스너에 연결된 기능(함수)
      - 이벤트가 잠지 되었을 때 수행할 동작을 작성하는 함수

Inline Event Model

    - HTML 요소에 직접 이벤트 리스너 ='이벤트 핸들러' 형식으로 작성 
    - 최근엔 사용 빈도가 낮아짐
    - 인라인 이벤트 모델 작성시
      이벤트 핸들러 () 안에 this 작성 시 핸들러 함수로 이벤트가 발생된 요소가 전달

고전 Event Model

    - HTML에 이벤트 관련 코드를 직접 작성하지 않고
      JS 에서 DOM 을 이용해 이벤트가 처리되기 원하는 요소가
      가지고 있는 고유 이벤트 리스너에 이벤트 핸들러를 대입하는 형식

    * 단점 : 하나의 요소에 동일한 이벤트 리스너/핸들러 작성불가
      (마지막에 대입된 핸들러로 덮어쓰기)

표준 Event Model (요소.addEventListener())

    - W3C(웹 표준 지정 단체)에서 공식적으로 지정 리벤트 모델
    - 고전 이벤트 모델의 단점
      -> 이벤트 리스너가 종류별로 1개
    - 고전 이벤트 모델의 장점
      -> 이벤트 리스너를 원하는 만큼 추가
    
    [작성법]
